BEFORE

Pain Points

每次產品上線前，PM 都要面對這些重複性高、容易出錯的文件工作

📄

PRD 撰寫耗時

散亂的 Google Docs、Slack threads、PDF 需手動整理成結構化 PRD，每次花費 2-4 小時

🌐

翻譯流程瑣碎

EN/TH/JA 三語翻譯需手動對照術語表，review request 手動發 Slack 給不同 reviewer

🧪

測試追蹤困難

Feature Spec 中的 test cases 散落各處，無法自動產出腳本或追蹤結果

📢

Release Note 重工

每次從 PRD 手動摘要，重新排版 Internal Update，手動貼 Slack

SOLUTION

What We Built

4 個 Claude Code Skills，加速產品開發文件產出

1
/prd — User Scenario → Deep Research（codebase + reqflow + 競品）→ 完整 PRD + Feature Spec + Test Cases，一條龍產出
2
/test-case — 解析 test cases → Playwright 腳本 + AI 瀏覽器測試 + 測試報告 + Asana bug task
3
/release-note — PRD → Internal Update → 寫回 PRD Doc + 直推 Slack（含 thread 拆分）
4
/translate — 用戶填完 zh-TW → AI 自動翻譯 EN/TH/JA 寫回同一份 Sheet → 推 Slack review（附 PRD）→ Reviewer 確認後 locales-publish 自動更新 staging & production

QUICK START

How to Start

五步驟開始使用

1

Clone GitHub Repo

git clone https://github.com/Lydia47/pm-release-accelerator.git

2

安裝 Skills 到 Claude Code

cd pm-release-accelerator && cp -r .claude/commands/* ~/.claude/commands/

3

安裝 & 認證 gws CLI

npm install -g @googleworkspace/cli
gws auth setup → gws auth login -s drive,sheets,docs

4

開始使用

在 Claude Code 輸入 /prd interactive，貼入 user scenario 即可一條龍產出完整 PRD

5

下游串接

拿到 PRD Doc ID 後 → /release-note <DOC_ID> → /translate <product> auto
