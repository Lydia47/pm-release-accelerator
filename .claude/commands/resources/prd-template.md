### **Metric Goal**

* Overall Goal:   
  * Enable PMs to analyze adoption patterns and link them to HubSpot data for insights into post-tracking performance.  
    *   
  * Details:  
    * PM wants to identify the most common product feed upload method, along with the user's organization and email.   
    * 

### **Requirements Schema Definition**

* Environment:   
  * Production (TW/TH)  
  * Production (JP)  
* [Integration Example Data](https://docs.google.com/spreadsheets/d/1XgnKGfkrt8m9nWeLTVXpAmv4xdCft2JtUymmI-zHAmA/edit?gid=1920991504#gid=1920991504)   
* Note: The event schema will follow engineering needs. What’s listed below is just a starting point for required fields.

| Field Name | Field Description | Required | Note | Example Value |
| ----- | ----- | ----- | ----- | ----- |
| **event\_action** | Describes the product feed action type (create, update). | ✅ Yes | Used for funnel or retention tracking. | create, update, archive, restore |
| **feed\_name** | The name assigned by the user when creating the feed. | ✅ Yes | Helps identify feed lifecycle and reuse pattern. | Summer\_2025, Autumn\_Catalog, Winter\_Store |
| **org\_name** | Organization name associated with the feed. | ✅ Yes | Stored as a user-level property for aggregation by org. | FamilyMart\_TW, Krungsri\_Simple, Nestle\_JP |
